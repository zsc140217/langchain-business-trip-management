## 1. 向量数据库选型

### 1.1 名字发音和记忆技巧

#### FAISS（读音：菲斯 / 费斯）
- **记忆法**：**F**ast（快）+ **AI** + **SS**earch = "快速AI搜索"
- **口诀**："**费**事吗？不**费**！超级**快**！"
- **形象记忆**：像"飞丝"一样快（内存操作）
- **适用**：原型开发、小规模（<10万条）

**为什么选它？**
```
✅ Meta（Facebook）开源，质量可靠
✅ 纯内存，速度极快（毫秒级）
✅ 免费，适合学习和原型
❌ 重启数据丢失（需手动保存）
```

#### Chroma（读音：克罗玛）
- **记忆法**：Chroma = 色度，想象"**彩色数据库**"
- **口诀**："**彩色**数据**留**得住"（持久化）
- **形象记忆**：像彩色笔记本，数据会保存
- **适用**：本地开发、需要持久化

**为什么选它？**
```
✅ 嵌入式（不需要单独装服务）
✅ 自动持久化到 SQLite
✅ 免费开源
❌ 比 FAISS 慢，但数据不丢
```

#### Pinecone（读音：派恩空 / 松果）
- **记忆法**：Pine（松树）+ Cone（锥形）= 松果
- **口诀**："**松果**掉进**云**里"（云服务）
- **形象记忆**：像松果一样坚固可靠
- **适用**：生产环境、大规模

**为什么选它？**
```
✅ 完全托管（不用自己运维）
✅ 自动扩展（支持几十亿向量）
✅ 99.9% 高可用
❌ 贵（$70/月起）
```

#### Qdrant（读音：夸德兰特 / 象限）
- **记忆法**：Quadrant = 象限（数学术语）
- **口诀**："**象限**分得**清**，性能**顶呱呱**"
- **形象记忆**：像坐标系的四个象限
- **适用**：高性能需求、自己部署

**为什么选它？**
```
✅ 性能极高（Rust 语言编写）
✅ 可以自己部署（省钱）
✅ 也有云服务版本
✅ 比 Pinecone 便宜
```

#### Milvus（读音：米尔沃斯）
- **记忆法**：Mil（千）+ vus，想象"**千军万马**"
- **口诀**："**千军万马**齐上阵，企业级的**大阵仗**"
- **形象记忆**：像大军团，适合企业级
- **适用**：企业级、超大规模

**为什么选它？**
```
✅ 专为企业设计
✅ 分布式架构（可以多台机器）
✅ 功能最全
❌ 复杂，小团队用不上
```

## 2. 嵌入模型选型

### 2.1 什么是嵌入模型？（白话解释）

**比喻**：
- 嵌入模型 = 翻译官
- 把文字翻译成数字（向量）
- 相似的文字 → 相似的数字
- 例如："猫"和"喵"的数字很接近

### 2.2 核心嵌入模型详解

#### OpenAI text-embedding-3-small（性价比之王⭐⭐⭐）

**名字记忆**：
- 口诀："**小三**最**实惠**"（第3代小模型）

**训练数据分析**：
```
训练语料来源：
├── 英文为主（70%）
│   ├── 维基百科
│   ├── 书籍、论文
│   └── 网页爬虫
├── 多语言（30%）
│   ├── 中文：★★★★☆（较好）
│   ├── 其他语言：100+ 种
│   └── 训练时间：2023年

效果评分（MTEB 基准）：
├── 英文理解：★★★★☆（62.3分）
├── 中文理解：★★★★☆（还不错）
└── 多语言：★★★★☆

成本对比：
├── 价格：$0.02/1M tokens
├── 维度：1536
└── 性价比：⭐⭐⭐⭐⭐
```

**什么场景用？**
- ✅ 英文为主的项目
- ✅ 预算有限
- ✅ 需要稳定 API

#### HuggingFace bge-m3（中文开源之王⭐⭐⭐）

**名字记忆**：
- bge = **B**AAI **G**eneral **E**mbedding（北京智源通用嵌入）
- m3 = **M**ulti-lingual + **M**ulti-granularity + **M**ulti-functionality
- 口诀："**北京造**的**三多**模型"

**训练数据分析（重点）**：
```
训练语料详解：
├── 中文语料（40%） ⭐⭐⭐⭐⭐
│   ├── 百度百科（全量）
│   ├── 维基百科中文版（全量）
│   ├── 知乎问答（精选）
│   ├── 豆瓣评论（精选）
│   ├── 中文新闻（人民日报、新华社等）
│   ├── 中文书籍（现代文学、技术书籍）
│   └── 中文论坛（天涯、贴吧等）
├── 英文语料（40%）
│   ├── 英文维基百科
│   ├── 英文书籍
│   └── 学术论文
└── 其他语言（20%）
    └── 100+ 种语言

中文优势详解：
├── 理解多义词
│   ├── "打"：打车、打人、打球、打电话
│   ├── "意思"：这个意思真有意思
│   └── "行"：这行不行？我是干这行的
├── 理解中文语境
│   ├── "马马虎虎"（不是马和老虎）
│   ├── "差不多"（口语化程度理解）
│   └── "忽悠"（东北方言）
├── 理解中文俗语
│   ├── "三顾茅庐"（典故）
│   ├── "画龙点睛"（成语）
│   └── "吃瓜群众"（网络用语）
└── 理解中文结构
    ├── "把"字句：把书放桌上
    ├── "被"字句：被老师批评了
    └── 省略主语：去哪儿？（谁去？）

为什么免费：
├── 开源项目（北京智源研究院）
├── 可以本地部署
├── 不需要 API 费用
└── 国家科研支持

效果评分：
├── 中文理解：★★★★★（最强）
├── 英文理解：★★★★☆
├── MTEB 分数：63.0
└── 维度：1024
```

**什么场景用？**
- ⭐⭐⭐ 纯中文项目（**最佳选择**）
- ⭐⭐⭐ 预算为 0
- ⭐⭐⭐ 需要本地部署（数据隐私）
- ⭐⭐⭐ 政府、银行等不能用云服务

**实际效果对比**：
```
测试：搜索"苹果手机怎么截屏？"

OpenAI small 结果：
1. iPhone screenshot methods
2. 苹果手机截屏教程
3. Apple device screen capture

bge-m3 结果：
1. 苹果手机截屏教程
2. iPhone 截图方法
3. 如何给苹果手机截屏

结论：bge-m3 更理解中文表达习惯
```

## 3. LLM 模型选型

├── 训练数据包含大量工具调用示例
├── 输出 JSON 格式极其稳定
├── 参数提取准确率 > 99%
└── 错误率 < 1%（远低于其他模型）

成本：
├── 输入：$5/1M tokens

- ⭐⭐⭐ 智能客服（出错不起）

## 5. 面试答题模板

### Q: 如何选择嵌入模型？

**答题框架**（30秒）：
> "选择嵌入模型主要看三个维度：语言、预算、训练数据。纯中文项目用 bge-m3，因为它 40% 的训练语料是中文，理解中文俗语、多义词最好，而且免费可本地部署。跨境电商用 Cohere embed-v3，专门优化了多语言和中英混合查询。英文项目用 OpenAI small，性价比最高。"

### Q: 什么时候用 Qwen-Max？

**答题框架**（20秒）：
> "Qwen-Max 适合纯中文内容生成项目。它 60% 的训练语料是中文，包含淘宝、知乎、微博等真实中文数据，生成的文案最地道。比如电商商品描述、中文营销文案。成本比 GPT-4o 便宜 70%，但工具调用不可靠，不适合需要调用 API 的场景。"
